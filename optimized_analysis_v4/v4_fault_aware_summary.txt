Isolation Forest V4 fault-aware 优化说明

1. V4 的目标

V4 延续 V3 的平衡思路，但不再只看 overall 指标。
本版重点解决的问题是：
- pause 类型故障命中率长期低于 kill
- 更严格的平衡配置虽然提升 precision，但会过度牺牲 pause 命中
- 因此参数选择需要显式考虑 fault_type 差异

2. V4 的核心变化

- 保留 V3 的基础参数与后处理联合搜索框架
- 在验证集上新增 kill_hit_rate 与 pause_hit_rate
- 选择主配置时优先考虑 pause_hit_rate，而不是只看总 event_hit_rate
- 同时继续约束 avg_anomaly_ratio <= 0.4

3. 覆盖率背景

- validation 严格故障窗口覆盖率 = 0.0639
- validation 扩展 +/-120 秒后的覆盖率 = 0.3151
- 说明异常邻域仍然存在，但 V4 更关注不同故障类型之间的检测公平性。

4. 两类代表配置

- 高覆盖配置: n_estimators=300, contamination=0.12, postprocess=streak_2_smooth_5_vote_2, event_hit_rate=1.0, pause_hit_rate=1.0, anomaly_ratio=0.7187
- fault-aware 主配置: n_estimators=100, contamination=0.08, postprocess=streak_3, event_hit_rate=0.7778, pause_hit_rate=0.75, anomaly_ratio=0.3495

5. V4 最终推荐配置

- n_estimators = 100
- contamination = 0.08
- feature_set = full_features
- postprocess = streak_3
- min_streak = 3
- smooth_window = 1
- vote_threshold = 1

9. 当前结论

- V4 不是重新追求全局高覆盖，而是让 pause 不至于在选型阶段被系统性牺牲。
- 如果 V4 的 pause_hit_rate 提升且 F1 仍稳定，就说明 fault-aware 选型是有效的。
- 如果 pause 提升但 precision 明显恶化，下一步应做更细的 pause 专项特征或窗口策略。
